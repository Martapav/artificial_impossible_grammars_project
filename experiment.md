You are assisting with the setup of a research computing environment for a master's thesis in computational linguistics and mechanistic interpretability. The project is conducted at CIMeC, University of Trento. All code, documentation, and comments are in English.

The project investigates whether transformer language models encode hierarchical versus linear grammatical structure in distinct internal representations. The experimental design uses four formally defined artificial grammars to control the training signal precisely and support causal claims about what structural properties are learned. 

# Grammar Implementation Brief
## Artificial Grammar Learning Paradigm — Corpus Generator

This document contains all information needed to implement the four grammar corpus generators (H, H′, L′, P). Read it in full before writing any code.

---

## 1. Shared Typological Backbone

These constraints apply to all four grammars without exception.

- Non pro-drop: subject must always be overtly expressed.
- Morphology is fully analytical: 1:1 morpheme-to-meaning, 1:1 morpheme-to-word. Every morpheme is a separate space-delimited token.
- Basic linear order: SVO. All syntagms are head-first.
- The language is prepositional (prepositions precede their complements).
- No case morphology, no passivization, no moods, no morphophonological alternations.
- Morpheme order within a word-form: lexical morpheme first, inflectional morpheme(s) immediately following.
- Inflectional order within a word: for INFL1-bearing items, number morpheme precedes gender morpheme; for INFL3-bearing items, tense morpheme follows the lexical item (and CAT3AUX if present).
- Sentence production capacity: unbounded. No hard upper limit on the number of constituents per sentence. Sentence length will approximate a natural language distribution; this is controlled externally at corpus-construction time, not by the grammar rules themselves.

## 2. Lexicon

### 2.1 Alphabet

36-symbol alphanumeric alphabet: lowercase Latin letters a–z (26) plus digits 0–9 (10). Letters are reserved exclusively for lexical morphemes; digits are reserved exclusively for inflectional morphemes. These two sets never overlap.

### 2.2 Lexical pools by category

Each category has a non-overlapping dedicated letter set. Lexical items are strings over that category's letter set, generated by within-category permutation-with-repetition up to a fixed length bound. Inter-categorical homonymy is excluded by construction.

| Category | Letter pool | Notes |
|----------|-------------|-------|
| CAT1 | {a, b, c, d, e} | |
| CAT1PRON | {m} = Pron, {mm} = Refl | Exactly two items; both inflect for gender and number. The pronominal system is kept as simple as possible: no person distinction is implemented. |
| CAT2 | {f, g, h, x} | |
| CAT3 | {i, j, k, l} | |
| CAT3AUX | {n} | Exactly one item |
| CAT4 | {u} = singular, {v} = plural | Two items; number-marked intrinsically; no inflectional morphemes |
| CAT5 | {o, p, q} | |
| CAT6 | {r, y, z} | |
| CAT8 | {w} | Exactly one item |
| CAT9 | {s, t} | Exactly two items |

### 2.3 Inflectional pools

| Category | Parameters | Values | Morpheme pool |
|----------|------------|--------|---------------|
| INFL1 | number (2 values), gender (2 values) | ±sg, ±m | {1, 2, 3, 4} — assign one digit per value-combination |
| INFL3 | tense (3 values) | present, past, future | {5, 6, 7} |

INFL1 is expressed as two separate morpheme tokens (number morpheme, then gender morpheme) immediately following the lexical morpheme of any INFL1-bearing item (CAT1, CAT2, CAT3 in agreement position). INFL3 is expressed as one morpheme token immediately following the lexical morpheme of CAT3 or CAT3AUX.

Null morpheme: represents gaps in the string (e.g., trace of a moved element). Not lexicalized at generation time; used in annotation/evaluation only.

### 2.4 Lexical features

- **CAT1 [±cnt]**: countable ([+cnt]) items require a CAT4 determiner and admit CAT2 modifiers; non-countable ([-cnt], proper-noun-type) items take no CAT4 and no CAT2.
- **CAT3 [±tr]**: transitive ([+tr]) items require an object Type1 syntagm; intransitive ([-tr]) items take no object.
- **CAT6 attachment partition**: the three CAT6 items are partitioned into two non-overlapping pools at lexicon-generation time. Two items belong to the CAT1-selecting pool (PP-under-NP; nominal head arguments) and one item belongs to the CAT3-selecting pool (PP-under-VP; adverbial adjunct of CAT3). This assignment is fixed for the entire paradigm and is not drawn randomly.

### 2.5 Frequency distribution

Lexical types are assigned Zipfian frequency ranks prior to corpus construction, based on functional class (closed-class/function items rank higher than open-class/content items). The corpus generator samples item occurrences from these rank-frequency distributions. Morphological complexity (compounding length) is orthogonal to frequency assignment.

## 3. Lexical Generation Algorithm

1. For each category, select a target lexicon size within the cardinality bound imposed by the permutation-with-repetition formula over the category's letter pool.
2. Generate items deterministically using a seeded RNG so that lexicons are reproducible.
3. Assign binary lexical features ([±cnt] to CAT1 items, [±tr] to CAT3 items) by seeded random draw.
4. Assign CAT6 items to pools: two items to the CAT1-selecting pool, one item to the CAT3-selecting pool. This is a fixed partition, not a random draw.
5. Assign Zipfian frequency ranks to all items within each category.
6. Export the full lexicon as a structured object (e.g., JSON) with all features recorded. The lexicon is grammar-agnostic; all four grammars import from it.

## 4. Syntactic Categories and Syntagm Types (Hierarchical Grammars: H, H′, L′ base)

### 4.1 Syntagm types

**Type0 (Sentence)**: the top-level node. Expands as: Type1 + Type2. Corresponds to the S node.

**Type1 (Nominal projection)**: headed by CAT1. Two sub-schemas:
- If CAT1 is [+cnt]: `CAT1 INFL1n INFL1g CAT4 (CAT2 INFL1n INFL1g)*`
- If CAT1 is [-cnt]: `CAT1 INFL1n INFL1g`

After the core nominal, Type1 may take:
- Zero or more Type3 dependents (PP adjuncts or arguments; count unbounded in the base grammar but rate-controlled at corpus construction time).
- Zero or more Type4 dependent (relative clause), positioned finally within Type1.

**Dependent Type1 (T1dep)**: appears in non-subject positions (objects of Type2, complements of Type6). Has the same internal structure as Type1.

**Type2 (Verbal projection)**: headed by CAT3. Two schemas, simple and compound tense:
- Simple: `CAT3 INFL1n INFL1g INFL3 (CAT5)* (Type3)* (T1dep)*`
- Compound: `CAT3 CAT3AUX INFL1n INFL1g INFL3 (CAT5)* (Type3)* (T1dep)*`

The compound-tense form: CAT3AUX immediately follows CAT3; inflectional morphemes (INFL1n, INFL1g, INFL3) follow CAT3AUX. CAT5 adverbs follow inflectional morphemes. The object T1dep is obligatory if CAT3 is [+tr], absent if [-tr]. Type3 adjunct at this level is a PP-under-VP (CAT6 from the CAT3-selecting pool), and is optional regardless of transitivity.

Agreement enforcement: the INFL1n and INFL1g values on CAT3 (or CAT3AUX) must match those of the subject Type1's CAT1 head.

**Type3 (Prepositional phrase)**: headed by CAT6. Expands as: `CAT6 T1dep`. CAT6 is non-inflecting. Type3 attaches either (a) as a dependent of a CAT1 head within a Type1 syntagm (PP-under-NP, uses a CAT6 from the CAT1-selecting pool) or (b) as an adjunct of a CAT3 head within a Type2 syntagm (PP-under-VP, uses a CAT6 from the CAT3-selecting pool).

**Type4 (Subordinate clause)**: headed by CAT9. Expands as: `CAT9 Type0`. Attaches exclusively as a dependent of a CAT1 head within a Type1 syntagm (relative clause only). 

Embedding: Type4 introduces a complete Type0, which may itself contain Type1 nodes bearing further Type4 dependents. Embedding depth is not hard-capped by the grammar rule itself (production is unbounded); depth is controlled externally by the length-distribution sampler at corpus construction time.

### 4.2 Linear surface order

For a declarative sentence the resulting surface order is (schematically):

```
[CAT1 INFL1n INFL1g (CAT4) (CAT2 INFL1n INFL1g)* (Type3)* (Type4)?]_subj
[CAT3 (CAT3AUX) INFL1n INFL1g INFL3 (CAT5)* (Type3)?]_verb
[CAT1 INFL1n INFL1g (CAT4) (CAT2 INFL1n INFL1g)* (Type3)*]_obj
```

## 5. Positional Grammar Base (Grammar P, and the positional backbone of L′)

In positional grammars there are no syntagms, no heads, no c-command, no dominance. All well-formedness conditions are stated in terms of category identity and ordinal position computed by a left-to-right scan. The positional rules produce surface strings maximally similar to hierarchical output except where transformation rules necessarily diverge.

### 5.1 Positional construction rules (applied in order, left-to-right scan)

1. **Base order**: CAT1 + CAT3 + CAT1 (subject, verb, object). Intransitive CAT3: omit object CAT1.
2. **CAT4 insertion**: for each CAT1 encountered in scan order, if that CAT1 is [+cnt], insert CAT4 immediately after its inflectional morphemes. CAT4 form determined by the number value of the CAT1's INFL1n.
3. **CAT3AUX insertion**: if compound tense, insert CAT3AUX immediately after CAT3 and before its inflectional morphemes.
4. **CAT2 insertion**: insert CAT2 items immediately after CAT1 (or after CAT4 if present). Maximum per CAT1: unbounded in grammar, rate-controlled externally.
5. **CAT5 insertion**: insert CAT5 items immediately after CAT3 (or after CAT3AUX if present). Maximum per CAT3: unbounded in grammar, rate-controlled externally.
6. **CAT6 insertion**: two positional behaviors depending on local context:
   - If a CAT2 is present anywhere after the most recently processed CAT1, insert CAT6 immediately after the first CAT2 encountered.
   - Otherwise insert CAT6 immediately after CAT1 (or CAT4 if present). If CAT5 is present and no CAT2, insert after CAT5; if no CAT5, insert after CAT3.
   - Each CAT6 must be immediately followed by a CAT1. Multiple CAT6 items each require their own following CAT1.
7. **CAT9 insertion**: CAT9 initializes a new substring. The new substring follows the same positional rules from step 1.If CAT9 is present and the scan reveals the presence of a CAT2, then the CAT9 must be positioned right after the last CAT2 it finds in the linear scan. If there is no CAT2, the same applies for CAT4. If there is no CAT4, then it must be placed right after CAT1. Maximum CAT9 substrings per sentence: unbounded in grammar, rate-controlled externally.
8. **Inflection**: INFL1n and INFL1g values for CAT3 are determined by counting to the CAT1 at subject position (position 1 in left-to-right scan) and copying its INFL1 values. The second CAT1 (object) takes independently drawn INFL1 values.

**Constraint on scan-based generation**: no production step may depend on prediction of items not yet encountered in the scan. Each step operates only on already-processed positions.

## 6. Grammar Definitions

### 6.1 Grammar H — Fully Hierarchical

All well-formedness conditions are stated over constituent structure. All four phenomena (binding, agreement, auxiliary movement, wh-movement) use their hierarchical formalizations (see Section 7). Grammar H is a context-free phrase-structure grammar.

### 6.2 Grammar P — Fully Positional

All well-formedness conditions are stated over surface position by category identity. Grammar P uses the positional backbone from Section 5 throughout. All four phenomena use their positional formalizations. Grammar P is a regular/finite-state grammar (counting grammar); it has no access to recursive embedding in the structural sense.

### 6.3 Grammar H′ — Hierarchical with an Irreducibly Positional Rule

Base architecture: identical to Grammar H (phrase-structure). All phenomena use hierarchical formalizations **except** the complex NP island constraint in wh-movement, which is replaced by the following positional blocking condition:

> For an item x targeted for wh-fronting: if the item **immediately preceding x** in the linear string belongs to CAT6, and the item **immediately preceding that CAT6** belongs to CAT1, then fronting of x is blocked.

This condition is computed by a left-to-right scan of the surface string. All other island constraints in H′ (subject-phrase extraction, adjunct island) are stated structurally, as in H.

### 6.4 Grammar L′ — Positional with an Irreducibly Hierarchical Rule

Base architecture: identical to Grammar P (positional, scan-based). All phenomena use positional formalizations **except** the complex NP island constraint in wh-movement, which is replaced by the following structural blocking condition:

> For an item x targeted for fronting: if x is dominated by a CAT6 projection that is itself dominated by a CAT1 projection (i.e., x is the complement of a CAT6 that is an argument of a CAT1 nominal head), then fronting of x is blocked.

This condition is evaluated over the **generation history**, not the surface string. The generator maintains a record of which item licensed each insertion and in which structural role. The block fires if and only if the record shows: x was inserted as the complement of a CAT6 item, and that CAT6 was inserted as a dependent of a CAT1 item (PP-under-NP configuration). Generation history makes no reference to surface position.

### 6.5 Divergence Cases (H′ vs. L′)

The H′ positional condition and the L′ structural condition are not equivalent. Two classes of divergence exist:

**Case 1 — CAT2 intervention**: A CAT2 modifier intervenes between the nominal head (CAT1_head) and the PP in the surface string:
```
... CAT1_head INFL1n INFL1g CAT2 INFL1n INFL1g CAT6 CAT1_x INFL1n INFL1g
```
- L′ blocks fronting of CAT1_x (structurally: CAT1_x dominated by CAT6 dominated by CAT1_head).
- H′ permits fronting (surface scan finds CAT2 before CAT6, not CAT1, so the positional condition is not triggered).
- Result: fronting grammatical in H′, ungrammatical in L′.

**Case 2 — CAT6 as adverbial adjunct (PP-under-VP)**: A CAT6 item functions as a CAT3-adjunct and surfaces immediately after the object CAT1:
```
... CAT3 INFL1n INFL1g INFL3 CAT1_obj INFL1n INFL1g CAT6 CAT1_x INFL1n INFL1g
```
- L′ permits fronting (structurally: CAT6 is dependent of CAT3, not of any CAT1; the structural condition does not apply).
- H′ blocks fronting (surface scan finds CAT1_obj immediately before CAT6, triggering the positional condition).
- Result: fronting grammatical in L′, ungrammatical in H′.

These two cases define the divergence items used in the generalization test set.

## 7. Syntactic Transformations

### 7.1 Anaphoric Binding

CAT1PRON items are introduced by a syntactic-lexical transformation: a Type1 syntagm (or CAT1 item) is substituted with a CAT1PRON item. Substitution is not optional — if the syntactic context licenses it, it must be instantiated.

CAT1PRON has two members: Pron (m) and Refl (mm), both inflecting for gender and number (INFL1 values carried over from the substituted CAT1). The pronominal system is kept as simple as possible: no person distinction is implemented. CAT4 and CAT2 items modifying the substituted CAT1 must be deleted, since CAT1PRON referential content is fully determined by the binding relation.

**Hierarchical formalization**

- **Principle A (Refl)**: Refl must be coindexed with a c-commanding antecedent within its local binding domain (= the minimal clause containing both Refl and a subject). The antecedent is a CAT1 (or Pron) that c-commands Refl within that domain.
- **Principle B (Pron)**: Pron must be free within its local binding domain. It must not be coindexed with any CAT1 (or Pron) that c-commands it within that domain. Coreference across clause boundaries is permitted.
- Intervening material within the clause does not affect binding; only c-command and domain membership matter.
- For reflexive substitution: the sentence must contain the same CAT1 item in both the antecedent position and the position to be substituted, within the same binding domain.
- For pronominal substitution: if coreference is present, the coreferential CAT1 must be in a different clause from the Pron.

**Positional formalization**

- **Principle A (Refl)**: Refl must occupy the position of the CAT1 (or CAT1PRON) immediately following CAT3 and its inflectional morphemes in the base string (i.e., direct object position). For Refl substitution, there must be a CAT1 in that position.
- **Principle B (Pron)**: Pron must not occupy the position immediately following CAT3 and its inflectional morphemes. For Pron substitution with coreference, no special positional condition constrains it; coreference is permitted as long as the Pron is not in direct object position (relative to the first CAT3 in left-to-right scan).
- In the positional grammar, the clause boundary for the purpose of Principle B is delimited by CAT9: the positionally-defined clause containing Pron runs from CAT9 (or sentence-start) to the next CAT9 (or sentence-end). Pron is free if no coindexed CAT1 occurs within that sub-string.

**CAT4/CAT2 deletion on substitution**: when the substitution step targets a CAT1 item at position i, the recognizer scans the immediately following positions and deletes any CAT4 and CAT2 items (plus their inflectional morphemes) occurring contiguously after that CAT1, before inserting CAT1PRON.

---

### 7.2 Auxiliary Movement (Question Formation)

The general rule: in an interrogative sentence, CAT3AUX fronts before the subject.

**Hierarchical formalization**: for an item x that is a CAT3AUX, if the sentence is interrogative, x is fronted to the position immediately preceding the subject constituent if and only if x belongs to the **matrix** Type0 syntagm. CAT3AUX items contained in Type4 dependents (subordinate clauses introduced by CAT9) are not eligible for fronting. The subject is the Type1 constituent that the CAT3 of the matrix Type0 agrees with.

String requirements: the sentence must contain at least one CAT3AUX in the matrix clause.

**Positional formalization**: for an item x that is a CAT3AUX, if the sentence is interrogative, the **first** CAT3AUX encountered in a left-to-right scan of the string is fronted to absolute string-initial position, regardless of which clausal domain it belongs to.

String requirements: the sentence must contain at least one CAT3AUX anywhere in the string.

---

### 7.3 Wh-Movement and Island Constraints

Wh-movement: a constituent is substituted by CAT8 (the wh-word) and displaced to the left periphery of the sentence (immediately preceding the subject in declarative order, i.e., sentence-initial position). Only Type1 syntagms (hierarchical) or CAT1 items (positional) can be fronted. No pied-piping. Wh-word substitution without fronting is not permitted.

**Hierarchical formalization**: fronting of a Type1 syntagm is blocked if its head CAT1 is contained in any of the following island structures:

1. **Subject-phrase extraction**: the targeted Type1 is itself the subject (matrix Type1 in Type0).
2. **Adjunct island**: the targeted Type1 is contained within a Type4 (relative clause).
3. **Complex NP island**: the targeted Type1 is contained within a Type3 that is itself an argument of a CAT1 head (PP-under-NP configuration). *(This condition is replaced in H′; see Section 6.3.)*

**Positional formalization**: fronting of a CAT1 item x is blocked if any of the following holds:

1. x is the second CAT1 to occur in the sentence (= object position in SVO base).
2. x is the first CAT1 occurring after a CAT9 (= subject of a subordinate clause).
3. x is the CAT1 immediately following a CAT6. *(This condition is replaced in L′; see Section 6.4.)*

If fronting is licensed, the item (Type1 or CAT1) is substituted with CAT8 and moved to sentence-initial position. The fronted item retains its inflectional morphemes (since CAT8 is a single non-inflecting particle that precedes the gap-bearing string).

String requirements: no special base structure is required; all blocking constraints are enforced during the syntactic transformation step.

## 8. Generation Procedure

### 8.1 Hierarchical grammar generation (H, H′, and hierarchical sub-system of L′)

Steps executed in order for each sentence:

1. **Transformation pre-selection**: draw whether a syntactic transformation will be applied, and if so which one (binding / auxiliary movement / wh-movement / none). Record this to constrain the base sentence's required structural elements (e.g., if auxiliary movement is selected, the base sentence must contain a CAT3AUX in the matrix clause; if binding is selected, the base sentence must contain two coreferential CAT1 items in the appropriate configuration).

2. **Base sentence generation**: generate the simplest admissible structure (no adjuncts, no expansions beyond what the transformation pre-selection requires). Choose a CAT1 subject (with [±cnt] feature determining CAT4 need) and a CAT3 (with [±tr] determining whether an object is needed).

3. **Inflection**: for each lexical item, assign inflectional values. Subject CAT1 receives freely drawn INFL1n and INFL1g. CAT3 (or CAT3AUX) receives INFL1n and INFL1g matching the subject; INFL3 drawn freely. Object T1dep receives independently drawn INFL1n and INFL1g. CAT2 modifiers agree with their CAT1 head.

4. **Obligatory non-inflecting item insertion**: scan the string and insert any required CAT4 items (determined by [±cnt] and INFL1n of the licensing CAT1). If compound tense, insert CAT3AUX immediately after CAT3 and reposition inflectional morphemes.

5. **First expansion (obligatory constituents)**: scan the string. For each potential head, draw whether to expand. If expanding, draw the number and type of dependents. Insert lexical morphemes head-first, then inflectional morphemes in licensed positions with values determined by the head. Iterate over all base-sentence items.

6. **Second expansion (adjuncts)**: scan and identify admissible adjunct attachment sites. For each site, draw (rate-bounded for length control) whether to attach an adjunct, and construct it. This is where Type3 PP adjuncts and Type4 relative clauses are added.

7. **Syntactic transformation**: apply the pre-selected transformation function to the completed string.

### 8.2 Positional grammar generation (P, and positional sub-system of H′)

Steps:

1. **Transformation pre-selection**: same as hierarchical step 1.

2. **Base sentence generation**: generate the base SVO structure (CAT1 CAT3 CAT1 for transitive; CAT1 CAT3 for intransitive). No inflectional values yet.

3. **Inflection**: scan left-to-right. Insert INFL1n and INFL1g immediately after the first CAT1 (subject). Insert INFL1n and INFL1g (copied from subject) and INFL3 immediately after CAT3 (or after CAT3AUX if compound). Insert INFL1n and INFL1g for the second CAT1 (object) after its lexical morpheme.

4. **Obligatory non-inflecting item insertion**: scan and insert CAT4 where required by [+cnt] CAT1 items. If compound tense, insert CAT3AUX immediately after CAT3 and reposition inflectional morphemes.

5. **First expansion (CAT2 and CAT5)**: insert CAT2 modifiers and CAT5 adverbs following the positional rules in Section 5.

6. **Second expansion (CAT6 and CAT9)**: insert CAT6 and CAT9 following the positional rules in Section 5. When CAT9 introduces a new clause, treat the new substring as a fresh sentence and apply steps 2–5 recursively to it.

7. **Syntactic transformation**: apply the pre-selected transformation.

### 8.3 Generation history tracking (required for L′ and recommended for all grammars)

The generator maintains a structured record for each sentence. For every non-base item added, the record stores: the item's category, the item that licensed its insertion, and the role in which it was inserted (e.g., "T3 argument of CAT1_head at position i", "T3 adjunct of CAT3 at position j", "T4 relative clause of CAT1 at position k"). This record is used to evaluate the L′ structural blocking condition (Section 6.4) and to construct the generalization test set.

## 9. Data Splits and Corpus Construction

### 9.1 Training corpus

- Size: 100,000 sentences per grammar.
- Distribution: 70% neutral sentences (no transformation); 10% per phenomenon (anaphoric binding, auxiliary movement, wh-movement with island constraints). Counts hold per grammar.
- Co-occurrence of multiple phenomena in a single sentence: permitted at a low, fixed rate.
- All data unannotated: no transformation tags, grammaticality labels, or structural metadata in the training string.
- Lexical and morphological distributions anchored by the Zipfian frequency assignment.

### 9.2 In-distribution test set

- Constructed by the same procedure as the training set.
- Contains minimal pairs: each well-formed string is paired with an ungrammatical variant differing at exactly one position critical to the target construction.
- Grammaticality judgment accuracy on this set is the primary learnability metric.

### 9.3 Generalization test set

- Probes structural generalization to configurations not represented in training.
- For Grammars H and P: varies embedding depth and dependency length.
- For Grammars H′ and L′: the critical items are the divergence cases from Section 6.5 (strings where the positional and structural rules make opposite predictions). Items are included only if the generation history confirms divergence; items where both rules agree may serve as controls.

### 9.4 Minimal pair test set

- Probes grammaticality judgements and robustness in assessing negative examples.
- Consisting in minimal pairs targeting both syntactically transformed sentences and neutral (base order) sentences.

### 9.4 Contamination prevention

Base sentences are assigned to exactly one split (train / dev / test / generalization) before any transformations are applied. Test and generalization items are derived exclusively from base sentences absent from training.

---

## 10. Tokenization

Tokenize by splitting on the separator (whitespace or `#` depending on implementation convention). Each token corresponds to exactly one morpheme: either one lexical morpheme or one inflectional morpheme. No multi-morpheme tokens. No subword tokenization. The vocabulary is closed and known in advance from the lexicon.

---

## 11. Key Implementation Notes

- All random draws use a seeded RNG. The seed must be recorded and reproducible.
- The lexicon is grammar-agnostic and generated once. All four grammars import from it.
- Each grammar's generator imports the lexicon and applies its own set of syntactic rules.
- The positional generator must never make forward-looking decisions: each insertion step operates only on the string state after all prior insertions.
- The hierarchical generator operates on a tree; all transformations are applied on the tree and the surface string is read off at the end.
- For L′: the generation history record must be maintained from the moment the first adjunct/complement is added and must be complete before the transformation step.
- Transformation step for all grammars: applied after the string is fully generated (all expansions complete). The pre-selection at step 1 only constrains which base structures are required; it does not pre-build the transformation — the transformation function is called on the completed string.

## Corpus Construction

For each grammar, three data splits are produced. The training corpus and the in-distribution test set are generated by the same procedure. A separate generalization test set contains structurally complex or marked configurations not represented in the training distribution, specifically configurations in which a linear heuristic and the correct hierarchical rule make divergent predictions. For H-prime and L-prime, generalization items are designed to probe each rule type independently. For Grammar P, structural complexity is varied along embedding depth and dependency length.

### Grammar variants

- **H**: phrase-structure, all rules structural.
- **P**: counter-automaton, all rules positional/count-based.
- **H′**: H base. Extraction from a PP argument of a nominal head governed by a positional rule: fronting blocked iff the item immediately preceding the target is CAT6 and the item before that is CAT1. All other islands as in H.
- **L′**: P base. Same phenomenon governed by a structural rule: fronting blocked iff the target is dominated by a CAT6 projection itself dominated by a CAT1 projection. Evaluated on the generation history, not the surface string.

The H′ and L′ rules are non-equivalent: divergent predictions arise (i) when CAT2 intervenes between the nominal head and the PP (L′ blocks, H′ permits), and (ii) when CAT6 functions as a CAT3 adjunct surfacing after the object CAT1 (H′ blocks, L′ permits).

### Generation

Minimal sentence: SVO with non-compound CAT3, optional CAT4 conditioned on CAT1 type. Phenomenon proportions fixed in advance; co-occurrence permitted at lower frequency. Wh-movement and auxiliary movement mutually exclusive within an item.

Hierarchical pipeline: base sentence → inflection (head-driven, dependent inheritance) → obligatory non-inflecting items (CAT4, CAT3AUX) → first expansion (obligatory constituents) → second expansion (adjuncts: PPs, coordinated structures) → syntactic transformation.

Positional pipeline: base sentence → inflection inserted at fixed string positions (2,3,5,6 for INFL1; 7 for INFL3; 9,10 for second CAT1) → obligatory non-inflecting items → first expansion (CAT2, CAT5) → second expansion (CAT6, CAT7; CAT7-conjoined sentences treated as fresh, CAT7 hard delimiter) → syntactic transformation.

Transformations: pronominal substitution (CAT1-headed constituent stripped and replaced by inflected CAT1PRON, eligibility per binding rules); auxiliary movement; wh-movement (CAT1-headed target fronted, CAT8 prefixed, gap signaled by null morpheme; islands evaluated pre-movement). For L′, the structural island filter consults the generation history record (licensor and role per non-base item), not the surface string.

## Technical Requirements

The project requires a grammar generation pipeline that defines rule sets per grammar, automates string generation, applies slot-controlled sampling to enforce distributional balance, and produces the three-component data split described above. It requires small transformer models trained from scratch using PyTorch with no pretrained weights. The DAS implementation uses pyvene (Wu et al., 2023); install from source with: pip install git+https://github.com/stanfordnlp/pyvene.git. An evaluation pipeline covers all Phase 1 and Phase 2 metrics.

## Environment Setup

Set up a Python virtual environment using Python 3.10 or later. Install: torch (verify CUDA compatibility first if a GPU is available), pyvene (install from source: pip install git+https://github.com/stanfordnlp/pyvene.git), numpy, scipy, pandas, matplotlib, and jupyter. Pin all versions in requirements.txt. The environment must be fully reproducible from requirements.txt alone.

## Plan Scope

When I issue the /plan command, produce a step-by-step plan covering: verifying Python version and available hardware; creating the virtual environment and installing dependencies in the correct order; initializing the directory structure above; creating stub files with docstrings for the grammar generation pipeline (grammar-agnostic infrastructure only, no construction-specific logic), the corpus construction pipeline (three-split output per grammar), and the DAS intervention scripts (with separate intervention set scaffolding for H-prime and L-prime); and writing a minimal README documenting the structure and reproduction steps.

Do not implement any construction-specific grammar logic at this stage. The pipeline stubs should be parameterized to accept a grammar type identifier and a rule type identifier (for the mixed grammars), but the rule logic itself is not scaffolded yet.
